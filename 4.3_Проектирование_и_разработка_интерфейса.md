# 4.3 Проектирование и разработка интерфейса

В данном разделе представлено подробное описание пользовательского интерфейса разработанной системы снабжения шиномонтажной мастерской. Интерфейс спроектирован с учётом требований эргономичности, интуитивной понятности и эффективности работы пользователей различных ролей.

## 4.3.1 Общие принципы проектирования интерфейса

При разработке интерфейса были применены следующие принципы:

1. **Единообразие дизайна** — все страницы выполнены в едином визуальном стиле с использованием общей цветовой схемы, типографики и компоновки элементов.

2. **Адаптивная навигация** — левая боковая панель навигации обеспечивает быстрый доступ ко всем основным разделам системы.

3. **Информативность** — каждая страница содержит заголовок, поясняющий её назначение, и контекстные элементы управления.

4. **Обратная связь** — система предоставляет пользователю визуальные уведомления о результатах выполненных действий.

5. **Минимизация ошибок** — формы ввода содержат валидацию данных, а критические действия требуют подтверждения.

### Цветовая схема

- **Основной цвет навигации**: градиент от #2c3e50 до #34495e (тёмно-синий)
- **Акцентный цвет действий**: градиент от #667eea до #764ba2 (фиолетовый)
- **Цвет успеха**: #27ae60 (зелёный)
- **Цвет предупреждения**: #e74c3c (красный)
- **Фон рабочей области**: #f5f6fa (светло-серый)

### Структура страниц

Каждая страница системы имеет единую структуру:
- **Левая боковая панель** — содержит логотип системы, основное меню навигации и кнопку выхода
- **Верхняя информационная панель** — отображает имя текущего пользователя и его роль
- **Основная рабочая область** — содержит контент страницы (таблицы, формы, виджеты)

## 4.3.2 Страница авторизации (Login Page)

**URL**: `/login/`  
**Шаблон**: `templates/registration/login.html`

### Назначение
Страница предназначена для аутентификации пользователей в системе. Доступ к функционалу системы предоставляется только после успешного ввода учётных данных.

### Визуальное описание

**Компоновка:**
- Центрированная форма входа на странице с градиентным фоном (от #667eea до #764ba2)
- Белый контейнер формы с закруглёнными углами (border-radius: 15px) и тенью
- Логотип и название системы в верхней части формы

**Элементы интерфейса:**

| Элемент | Описание |
|---------|----------|
| Заголовок «Система снабжения» | Крупный текст (26px), цвет #2c3e50 |
| Подзаголовок «Шиномонтажная мастерская» | Серый текст (14px), цвет #7f8c8d |
| Поле «Имя пользователя» | Текстовое поле с лейблом, обязательное для заполнения |
| Поле «Пароль» | Поле ввода пароля (тип password), обязательное для заполнения |
| Кнопка «Войти» | Градиентная кнопка на всю ширину формы с анимацией при наведении |
| Блок тестовых учётных записей | Информационная панель с данными для тестового доступа |

**Сообщения об ошибках:**
При неверном вводе данных отображается красный блок с текстом: «Неверное имя пользователя или пароль. Попробуйте снова.»

### Функциональность

1. **Ввод учётных данных** — пользователь вводит имя пользователя и пароль
2. **CSRF-защита** — форма содержит CSRF-токен для защиты от межсайтовой подделки запросов
3. **Перенаправление** — после успешного входа пользователь перенаправляется на страницу панели управления или на страницу, запрошенную до авторизации (параметр `next`)
4. **Отображение ошибок** — при ошибке аутентификации выводится сообщение об ошибке

|--------------|--------|-----|

Активный пункт меню выделяется фиолетовым градиентом.

**Основная рабочая область:**

**Верхняя часть:**
- Блок информации о пользователе с именем и ролью (например, «(Администратор)»)
- Заголовок страницы «Панель управления»
- Приветственный текст

**Виджеты:**

1. **Виджет уведомлений** (левая колонка)

     - Высокий приоритет: красный фон (#fdedec), красная полоса слева

   - При отсутствии уведомлений отображается пустое состояние с иконкой ✅

2. **Виджет материалов в цеху** (правая колонка, занимает 2 колонки сетки)
   - Таблица с материалами:
     - Столбцы: Материал, Количество, Мин. запас, Статус, Действие
     - Индикаторы статуса:
       - ✓ В норме (зелёный цвет #27ae60)
       - ⚠️ Низкий (красный цвет #e74c3c)
     - Бейджи количества:
       - Зелёный фон при нормальном запасе
       - Красный фон при низком запасе
   - Кнопка «Использовать» для каждого материала

### Функциональность

1. **Отображение уведомлений** — показывает последние события системы (новые заявки, низкий запас и т.д.)
2. **Мониторинг запасов цеха** — таблица с текущими остатками материалов
3. **Использование материала** — модальное окно для списания материала:
   - Поля: Материал (readonly), Доступное количество (readonly), Используемое количество, Примечание
   - Валидация: количество не может превышать доступное
   - Кнопки: «Отмена», «Подтвердить»

**JavaScript-функции:**
- `openUseModal(id, name, quantity, unit)` — открытие модального окна
- `closeUseModal()` — закрытие модального окна
- Обработчик отправки формы использования материала

## 4.3.4 Страница списка заявок (Requests List)

**URL**: `/requests/`  
**Шаблон**: `templates/procurement/requests.html`

### Назначение
Отображение списка всех заявок от сотрудников цеха на получение материалов со склада.

### Визуальное описание

**Верхняя панель:**
- Заголовок «📋 Заявки от сотрудников»
- Кнопка «➕ Создать заявку» (фиолетовая, градиентная)

**Таблица заявок:**

| Столбец | Описание |
|---------|----------|
| Номер заявки | Уникальный идентификатор заявки (жирный шрифт) |
| Заявитель | Имя пользователя, создавшего заявку |
| Описание | Краткое описание заявки (обрезается до 50 символов), список товаров |
| Дата создания | Дата и время создания в формате «дд.мм.гггг чч:мм» |
| Статус | Бейдж статуса (жёлтый «В обработке») |
| Действия | Ссылки «👁️ Просмотр» и кнопка «✅ Закрыть» |

**Список товаров в заявке:**
Отображается как маркированный список под описанием:
- • [Название товара] - [количество] шт.

**Пустое состояние:**
При отсутствии заявок отображается:
- Иконка 📋 (64px, полупрозрачная)
- Заголовок «Нет заявок»
- Текст «Заявки еще не созданы. Создайте первую заявку!»
- Кнопка «Создать заявку»

**Блок сообщений:**
Отображает уведомления об успешных операциях (зелёный фон) или ошибках (красный фон).

### Функциональность

1. **Просмотр списка заявок** — таблица со всеми заявками системы
2. **Переход к детальному просмотру** — клик на «👁️ Просмотр» открывает страницу details
3. **Закрытие заявки** — кнопка «✅ Закрыть» отправляет POST-запрос на закрытие
4. **Создание новой заявки** — кнопка в верхней панели перенаправляет на форму создания
5. **Пагинация** — при большом количестве заявок (реализуется на уровне view)

## 4.3.5 Страница создания заявки (Create Request)

**URL**: `/requests/create/`  
**Шаблон**: `templates/procurement/create_request.html`

### Назначение
Форма для создания новой заявки на получение материалов со склада цеха.

### Визуальное описание

**Верхняя панель:**
- Заголовок «📝 Создание новой заявки»
- Кнопка «← Назад к списку» (серая)

**Карточки формы:**

**Карточка 1: «Основная информация»**
- Поле «Общее описание заявки» — многострочное текстовое поле (textarea)
- Placeholder: «Опишите, для каких целей необходима данная заявка...»

**Карточка 2: «Материалы со склада»**
- Контейнер для динамических строк товаров
- Каждая строка содержит:
  - Выпадающий список «Материал со склада» (обязательное поле)
    - Отображает название и доступное количество
  - Поле «Количество» (числовое, min=1, обязательно)
  - Поле «Примечание» (текстовое, необязательное)
  - Кнопка удаления строки «✕» (красная)
- Кнопка «+ Добавить позицию» (зелёная)

**Нижняя панель действий:**
- Кнопка «✓ Создать заявку» (фиолетовая, отправка формы)
- Кнопка «Отмена» (серая, переход к списку)

### Функциональность

1. **Динамическое добавление позиций** — кнопка «+ Добавить позицию» создаёт новую строку с полями выбора товара
2. **Удаление позиций** — кнопка «✕» удаляет строку (минимум одна строка обязательна)
3. **Валидация формы**:
   - Все поля «Материал» должны быть заполнены
   - Все поля «Количество» должны быть ≥ 1
   - При ошибке поля подсвечиваются красной рамкой
4. **Отправка формы** — POST-запрос с данными:
   - `items_description` — общее описание
   - `products[]` — массив ID выбранных товаров
   - `quantities[]` — массив запрошенных количеств
   - `notes[]` — массив примечаний к позициям

**JavaScript-функции:**
- Обработчик клика «addItemBtn» — добавление новой строки
- Обработчик клика `.remove-item` — удаление строки
- Обработчик submit формы — предотправочная валидация

## 4.3.10 Страница создания заказа (Create Order)

**URL**: `/orders/create/`  
**Шаблон**: `templates/procurement/create_order.html`

### Назначение
Форма для создания нового заказа поставщику с выбором товаров и расчётом стоимости.

### Визуальное описание

**Верхняя панель:**
- Заголовок «➕ Создание заказа»
- Кнопка «← Назад к списку»

**Карточка 1: «Информация о заказе»**

| Поле | Тип | Обязательность |
|------|-----|----------------|
| Поставщик | Выпадающий список | Обязательно |
| Примечание | Textarea | Необязательно |

**Карточка 2: «Товары в заказе»**

**Таблица позиций:**

| Столбец | Описание |
|---------|----------|
| Товар | Выпадающий список с названием и остатком |
| Количество | Числовое поле (min=1) |
| Цена за ед. (руб.) | Числовое поле (step=0.01) |
| Сумма | Автоматический расчёт (qty × price) |
| Действие | Кнопка удаления строки «✕» |

**Подвал таблицы:**
- Строка «Итого:» с автоматическим расчётом общей суммы заказа

**Кнопки:**
- «+ Добавить позицию» (зелёная)
- «✓ Создать заказ» (фиолетовая)
- «Отмена» (серая)

### Функциональность

1. **Выбор поставщика** — выпадающий список с контактной информацией
2. **Динамическое добавление товаров** — неограниченное количество позиций
3. **Автоматический расчёт**:
   - При выборе товара автоматически подставляется цена
   - Сумма строки = количество × цена
   - Итоговая сумма = сумма всех строк
4. **Валидация формы**:
   - Поставщик обязателен
   - Минимум один товар в заказе
   - Количество ≥ 1, цена ≥ 0
5. **Удаление позиций** — удаление строки с пересчётом итогов

**JavaScript-функции:**
- `updatePrice(selectElement)` — автозаполнение цены
- `calculateRowTotal(inputElement)` — расчёт суммы строки
- `calculateGrandTotal()` — расчёт общей суммы
- `addRow()` — добавление новой строки
- `removeRow(button)` — удаление строки
- Обработчик submit формы с проверкой валидности

## 4.3.12 Технические особенности реализации интерфейса

### Используемые технологии

1. **HTML5** — семантическая разметка страниц
2. **CSS3** — стилизация с использованием:
   - Flexbox для компоновки
   - CSS Grid для виджетов dashboard
   - Градиенты и тени для визуальной глубины
   - Анимации transition для интерактивности
3. **Vanilla JavaScript** — клиентская логика без внешних зависимостей
4. **Django Templates** — серверный рендеринг с использованием:
   - Template tags (`{% %}`, `{{ }}`)
   - Наследование шаблонов (единая структура)
   - CSRF-токены для безопасности

### Адаптивность

Интерфейс адаптируется под различные размеры экрана:
- **Desktop (>768px)** — полная версия с боковой панелью
- **Mobile (<768px)** — боковая панель становится горизонтальной или скрываемой

### Доступность (Accessibility)

- Контрастность текста соответствует стандарту WCAG 2.1 AA
- Все интерактивные элементы имеют hover-состояния
- Формы содержат label для всех полей ввода
- Сообщения об ошибках визуально выделены

### Производительность

- Минимизация DOM-операций при динамическом добавлении элементов
- CSS-анимации вместо JavaScript для плавности
- Ленивая загрузка изображений (при наличии)
